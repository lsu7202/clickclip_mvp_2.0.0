# ClickClip 2.0.0 — 마스터 명세 (Master Spec)

> 단일 진실원천(Single Source of Truth). 이름·계약은 여기서 확정하고, 세부 명세는 이 문서를 참조만 한다.
> 작성 상태: **§0~§1 작성 완료 (리뷰 단계)** / §2~§9 작성 예정.

---

## §0 메타

### 0.1 목적
- 1.0의 기능을 계승하되, **화면 프레임 템플릿 / 이중 자막(자막1·자막2) / 비동기 동영상 분석 / AI 미디어(이미지·동영상 통합) / 스타일 프리셋**을 추가한다.
- 최우선 원칙: **식별자(이름) 흔들림 제거**. `userId ↔ user_id` 같은 변환 혼선을 구조적으로 차단한다.

### 0.2 1.0 대비 변경 요약
| 구분 | 1.0 | 2.0.0 |
|---|---|---|
| 화면 템플릿 | 없음 | **프레임 오버레이**(미디어 위·자막 아래 z-order), `my_templates` |
| 제목/폴더 | 고정 `clickclip` | **제목 입력 → 폴더명/파일명** |
| 자막 | 단일 | **자막1(내레이션/TTS) + 자막2(원본음성 분석)** |
| TTS 단위 | 장면 단위 | **자막1 줄(line) 단위**, 줄별 성우 가능 |
| 성우 | 단일(env 1개) | **다중 보이스맵**(env), 장면 기본값+줄별 예외 |
| AI 미디어 | 이미지/동영상 분리 | **통합 패널**(종류·참조·**스타일 프리셋**·비율) |
| 추천 키워드 | 있음 | **삭제**(gif/이미지 수동검색만) |
| 동영상 분석 | 없음 | **신규, 비동기**(샷 분할→장면 삽입, 음성→자막2) |
| 진행 표시 | — | **로딩 애니메이션만**(바·% 금지) |

### 0.3 문서/코드 규칙
- 서술은 한글, **식별자는 영문**.
- **레이어별 컨벤션 + 경계 변환**:
  - 프론트엔드(React): `camelCase`
  - **Wire(HTTP JSON): `snake_case`** ← 모든 API 요청/응답의 표준
  - 백엔드(Node): `camelCase` (경계에서 snake↔camel 변환)
  - AI 서버(FastAPI/Python): `snake_case` (wire와 동일 → 변환 0)
- 시간 단위 **µs(마이크로초)**, fps **30**, 캔버스 **1080×1920**.
- DB 없음. 프론트 상태 = SSOT. 백엔드 stateless(함수+파일+인메모리 job).

## §2 아키텍처

### 2.1 컨테이너 구성 (도커 필수)
| 서비스 | 스택 | 포트 | 비고 |
|---|---|---|---|
| `frontend` | React + Vite | 5173 | `camelCase` |
| `backend` | Node + Express | 4000 | `camelCase`, **ffmpeg/ffprobe 포함** |
| `ai_server` | FastAPI + Python | 8000 | `snake_case` |

- 호출 방향: 프론트 → 백엔드(HTTP, wire `snake_case`) → AI 서버(HTTP, `snake_case`).
- AI 서버는 프론트가 직접 호출하지 않음(백엔드 경유).
- 모든 환경변수는 **docker compose로 관리**.

### 2.2 변환 경계 (이름 흔들림 차단의 핵심)
변환은 **정확히 두 경계에서만** 일어난다. 그 외 모든 코드는 자기 레이어 컨벤션만 쓴다.
1. **프론트 API 클라이언트 인터셉터**: 요청 `camel→snake`, 응답 `snake→camel`. (단, `FormData` 업로드는 변환 제외)
2. **백엔드 미들웨어**: 요청 body `snake→camel`, 응답 `res.json` `camel→snake`.
3. **AI 서버**: 변환 없음(wire가 곧 `snake`).

```
React(camel) ─[client: camel→snake]→ HTTP(snake) ─[mw: snake→camel]→ Node(camel)
Node(camel) ─[client: camel→snake]→ HTTP(snake) ─────(변환없음)────→ FastAPI(snake)
```

### 2.3 상태/저장 모델
- **DB 없음.** 프론트엔드 상태(zustand)가 SSOT — 모든 작업 상태를 메모리에 보유.
- **백엔드 stateless**: 순수 함수 + 파일시스템(`WORKSPACE_DIR`) + **인메모리 job map**.
- 저장/불러오기/프로젝트 사이클 = 비범위. 새로고침 = 초기화.

### 2.4 비동기 job 모델 (동영상 분석 전용)
- 비동기는 **동영상 분석 한 곳**만. 나머지(분할/TTS/AI생성/검색/업로드/export)는 동기.
- 흐름: `POST /video-analysis` → 백엔드가 job 생성(`pending`) + `job_id` 즉시 반환 → 백그라운드 처리(`running`→`done`/`error`) → 프론트 `GET /jobs/:jobId` 폴링.
- 폴링 주기 권장 **1.5~2초**. 진행 표시는 **로딩 애니메이션만**(바·% 금지), 분석 중에도 다른 작업 가능.
- job map은 인메모리(프로세스 재시작 시 소실 — MVP 허용).

### 2.5 파일시스템 레이아웃
| 경로(env) | 마운트 | 소유/관리 | 용도 |
|---|---|---|---|
| `WORKSPACE_DIR` | named volume | 앱 | 생성 에셋(gif/image/ai/video/tts) |
| `MY_TEMPLATES_DIR` | bind | **사용자** | 프레임 PNG |
| `MY_SAMPLES_DIR` | bind | **사용자** | 참조 이미지 |
| `MY_STYLES_DIR` | bind | **사용자** | 스타일 `<name>.txt`+`<name>.jpg` |
| `MY_VOICES_DIR` | bind | **사용자** | 미리듣기 `<voice_id>.mp3` |
| `CAPCUT_TEMPLATE_DIR` | bind(ro) | 앱 | 골든 draft 스켈레톤 |
| `CAPCUT_DRAFT_ROOT` | bind | 사용자 환경 | export 산출물 위치 |

> bind mount 이유: 사용자가 Finder로 파일을 직접 떨궈 관리해야 하므로(템플릿/참조/스타일/보이스).

### 2.6 데이터 흐름 (요약)
1. **setup**: 템플릿·제목·언어·대본 입력 → `POST /scenes/split` → `scenes`(자막1 줄 포함). 대본 비면 빈 editor 진입.
2. **editor**: 줄 편집 / `POST /tts`(줄 단위) / 에셋(`/ai-media`·`/search`·`/upload`·`/assets/download`) / `POST /video-analysis`(job→폴링→장면 삽입).
3. **export**: `POST /export` → draft 2파일 + `assets/` + `subtitles.srt` zip, `CAPCUT_DRAFT_ROOT` 기준 경로 주입.


## §3 데이터 모델

> 예시는 프론트 SSOT 기준 `camelCase`. wire는 §2.2 경계에서 `snake_case`로 자동 변환된다.
> µs = 마이크로초. 모든 타이밍·길이는 µs 정수.

### 3.1 AppState (zustand, 프론트 전용)
```json
{
```

### 3.6 계산 규칙

**(a) 장면 길이 `durationUs`**
```
ttsTotalUs = Σ subtitle1Lines[i].tts.durationUs   // 생성된 줄만 합산
mediaUs    = media.durationUs (동영상) | null (이미지·gif)
DEFAULT_STILL_US = 3_000_000   // 정지 미디어 기본 3초 (env 가능)

fitToTts == true  → durationUs = ttsTotalUs>0 ? ttsTotalUs : (mediaUs ?? DEFAULT_STILL_US)
fitToTts == false → durationUs = mediaUs ?? (ttsTotalUs>0 ? ttsTotalUs : DEFAULT_STILL_US)
```

**(b) `fitToTts` 기본값 (출처별)**
- 대본 분할로 생긴 장면 → `true` (길이를 내레이션에 맞춤)
- 동영상 분석으로 삽입된 장면 → `false` (길이 = 샷 길이)

**(c) 장면 전역 시작 시각**
```
sceneStartUs(n) = Σ_{k<n} scenes[k].durationUs
```

**(d) 자막1 타이밍 (장면 내, 줄 순차 발화)**
```
lineStartUs(i) = sceneStartUs + Σ_{j<i} subtitle1Lines[j].tts.durationUs
자막1 표시구간 = [ lineStartUs(i), lineStartUs(i) + line.tts.durationUs ]
```
- 표시 단위 = 줄. `charTimings`는 향후 단어단위 분할 대비용(현재 미사용).

**(e) 자막2 타이밍**
- `startUs/endUs`는 분석 결과값(장면 로컬). export 시 `sceneStartUs` 더해 절대화.

**(f) `text ↔ ttsText` 바인딩**
- 줄 생성 시: `ttsText = text`, `ttsTextEdited = false`.
- `text` 수정 → `ttsTextEdited==false`일 때만 `ttsText`도 동일 갱신.
- `ttsText` 수정 → `ttsTextEdited = true`, 이후 `text` 변경은 `ttsText`에 미반영.

**(g) 번호 재정렬**
- `sceneNumber`·`lineNumber`는 1부터 연속. 추가/삭제/분리/병합 시 즉시 재번호.
- 동영상 분석 삽입: 대상 장면 N → 샷 M개로 치환 시 장면 N..N+M−1, 이후 장면은 +（M−1） 시프트.


## §4 화면 명세

> UI는 1.0 와이어프레임을 그대로 따르지 않고 **사용성 우선**으로 구성한다(§8 원칙). 좌표·픽셀이 아니라 동작·상호작용을 정의한다.

### 4.1 기본 설정 화면 (`step: 'setup'`)
| 요소 | 동작 |
|---|---|
| 프레임 템플릿 | `GET /templates` 드롭다운(썸네일). 기본 = 없음(`templateId: null`) |
| 제목 입력 | `title`. 폴더명/파일명 원본 (검증은 §7) |
| 언어 설정 | `ko`/`ja` 드롭다운, 기본 `ko`. 장면분할 프롬프트 분기 + 영상분석 번역언어 결정 |
| 대본 입력 | `scriptText` textarea |
| 계속하기 | 대본 有 → `POST /scenes/split` 후 editor. 대본 無 → 빈 editor 진입 |

### 4.2 메인 작업 화면 (`step: 'editor'`) — 3-pane
| 영역 | 내용 |
|---|---|
| **좌** | 선택 장면 **합성 미리보기**(미디어+프레임+자막 위치) · **총 길이** · **다운로드(export)** |
| **중** | 장면 카드 리스트(메인 스크롤) · 장면 추가(사이/위/아래) |
| **우** | 에셋 패널(선택 장면 컨텍스트, 상단 "장면 N 적용 중") |

- 장면 카드 클릭/미디어 클릭 → `selectedSceneNumber` 갱신 + 우 패널·좌 미리보기 동기화.

### 4.3 장면 카드 (Compact)
- **헤더**: `장면 N` · 길이=TTS 토글(`fitToTts`) · 음소거(`muted`) · 삭제(연결 에셋 포함)
- **미디어 썸네일**: 이미지/gif=`<img>`, 동영상=`<video muted>`. 클릭 → 에셋 패널 타깃.
- **성우 기본값**: 장면 `voiceId` 드롭다운(`GET /voices`, 미리듣기 ▶ = mp3 정적재생)
- **자막1 줄 스택**: 각 줄 = `text` 입력 + `발음▾`(펼치면 `ttsText` 편집) + 줄 성우 override(선택)
- **자막2**: 기본 숨김. 있거나 `자막2 보기` 토글 시 표시(색 구분).
- **툴바**: `＋자막` · `TTS 생성`(줄/전체) · ⏱`durationUs`

### 4.4 자막 편집 인터랙션 (자막1 기준)
| 입력 | 동작 |
|---|---|
| 텍스트 중간 ENTER | 커서 기준 줄 **분리**(앞=현재 줄, 뒤=새 줄) |
| 줄 맨앞 ENTER | 위에 빈 줄 추가 |
| 줄 맨끝 ENTER | 아래 빈 줄 추가 |
| 빈 줄 backspace | 줄 **제거** |
| 빈 위치 hover | `＋자막추가` 버튼 노출 |
- 키보드 외 **명시 버튼도 병행**(발견성). 분리/추가/제거 시 §3(g) 재번호.

### 4.5 장면 분리 / 병합
- **분리**: 자막 줄 사이 hover → 클릭. 분기 **윗부분** 줄·미디어는 현재 장면 유지, **아랫부분** 줄은 **새 장면**으로 내려가며 그 장면 미디어는 **빈 상태**(이후 에셋 지정).
- **병합**: 분리의 역연산. 인접 두 장면 병합 → 자막1·자막2 줄 이어붙임, **미디어는 위 장면 것 유지**(아래 장면 미디어 폐기).

### 4.6 에셋 패널 (5탭, 선택 장면에 적용)
| 탭 | 동작 |
|---|---|
| **gif** | 수동 검색바 → `GET /search/gif` 그리드 → 클릭 시 `media` 적용 |
| **이미지** | 수동 검색바 → `GET /search/image` 그리드 → 적용 |
| **AI 미디어** | 종류(이미지/동영상)·스타일 프리셋(썸네일)·참조 이미지·비율·**상황 입력** → `POST /ai-media/generate` → 결과 뷰어 → 클릭 적용 |
| **업로드** | 이미지/동영상 업로드 → `POST /assets/upload` → 적용 |
| **동영상 분석** | 영상 업로드 → `POST /video-analysis`(job) → 로딩 애니메이션 → 완료 시 **확인 모달**(4.7) |

- **추천 키워드 기능 없음**(gif/이미지는 수동검색만).
- AI 미디어: 스타일 선택 시 프롬프트 라벨이 "상황 입력"으로 변경, 비율 옵션은 모델 지원분만 동적 노출.

### 4.7 동영상 분석 확인 모달
- 분석 완료(`job.status='done'`) → **"감지된 샷 N개" 미리보기** 리스트(각 샷 썸네일 + 자막2 미리보기).
- **자막 적용 체크박스**(기본 ON): ON = 자막2 포함 삽입, OFF = 컷(장면)만 삽입(자막2 제외).
- `적용` → `targetSceneNumber` 위치에 장면 M개 삽입(§3 g). `취소` → 폐기.

### 4.8 로딩 표현 (확정)
- **로딩 애니메이션만**, 진행바·퍼센트 금지.
- **국소 표시**(전역 오버레이 금지) — 작업 위치에만:
  - 장면 분할(계속하기) / 줄·전체 TTS / AI 생성(결과 뷰어) / 검색 그리드 / 동영상 분석(해당 job 영역)
- 비동기 동영상 분석 중에도 나머지 편집 가능.

### 4.9 다운로드(export)
- `다운로드` → `POST /export` → zip. 파일명·폴더명 = `title`(검증/충돌처리 §7).
- 완료 시 **최종 폴더명·경로를 화면에 표시**.


## §5 기능 상세

> 모든 req/resp는 wire `snake_case`. 백엔드↔AI 내부 호출도 `snake_case`.
> 외부 API 스키마(Gemini/Typecast/fal/VI/Giphy/SerpAPI)는 **코딩 직전 재검증**(§6).

### 5.1 장면 분할 — `POST /scenes/split`
```jsonc
// req
{ "script_text": "...", "language": "ko" }
// resp
{ "scenes": [
  { "scene_number": 1, "subtitle1_lines": [ { "line_number": 1, "text": "..." } ] }
] }
```
- AI: Gemini. `language`로 프롬프트 분기(ko/ja). 규칙: **모든 장면·줄을 합치면 원문과 100% 일치**.
- 응답은 최소형(번호+text). 나머지 필드는 **프론트/백엔드가 기본값으로 하이드레이트**:
  `ttsText=text`, `ttsTextEdited=false`, `voiceId=null`, `tts=null`, `media=null`, `subtitle2Lines=[]`, `muted=false`, `fitToTts=true`.

### 5.2 TTS — `POST /tts` (자막1 줄 단위)
```jsonc
// 프론트→백엔드 req
{ "tts_text": "가수 강민경이 극찬한 횟집이라는데", "voice_id": "tc_voice_minsu", "language": "ko" }
// 백엔드→프론트 resp
{ "local_path": "tts/3-1.mp3", "audio_format": "mp3", "duration_us": 2100000,
  "char_timings": [ { "char": "가", "start_us": 0, "end_us": 90000 } ] }
```
- AI 서버: Typecast `with-timestamps`, `granularity="char"`, 언어 `TYPECAST_LANG_MAP`, 속도 `TTS_SPEED`. base64 반환.
- 백엔드: base64 디코드 → `WORKSPACE_DIR/tts/<sceneNumber>-<lineNumber>.mp3` 저장 → `local_path` 반환.
- `voice_id` 우선순위: 줄 `voiceId` → 없으면 장면 `voiceId`(프론트가 결정해 전송).
- **전체 생성** = 프론트가 줄마다 순차 호출(국소 로딩).

### 5.3 AI 미디어 — `POST /ai-media/generate`
```jsonc
// req
{ "media_type": "image", "style_id": "cinematic", "situation_text": "횟집 내부, 회 접시 클로즈업",
  "reference_name": null, "aspect_ratio": "9:16" }
// resp (= Asset)
{ "local_path": "ai/x.png", "source_type": "ai_image",
  "width_px": 1080, "height_px": 1920, "duration_us": null, "has_audio": false }
```
**모델 분기 (참조 유무 × 종류)**
| media_type | reference | 사용 env | fal 호출 |
|---|---|---|---|
| image | 없음 | `FAL_MODEL_IMAGE_T2I` | text→image |
| image | 있음 | `FAL_MODEL_IMAGE_REF` | 참조 기반 편집 |
| video | 없음 | `FAL_MODEL_VIDEO_T2V` | text→video |
| video | 있음 | `FAL_MODEL_VIDEO_I2V` | image→video |

- **프롬프트 합성(서버)**: `style_id` 有 → `최종 = <style.txt 내용> + "\n" + situation_text`. `style_id` 無 → `최종 = situation_text`(전체 프롬프트).
- `reference_name` → `MY_SAMPLES_DIR/<reference_name>` 로딩해 모델 입력.
- `aspect_ratio`는 모델 지원 집합만(프론트가 동적 제한). 결과 다운로드→ffprobe(동영상)→Asset.

### 5.4 검색 — `GET /search/gif` · `GET /search/image`
```jsonc
// GET /search/gif?q=...   (Giphy)   /   GET /search/image?q=...  (SerpAPI)
{ "results": [ { "url": "https://...", "width_px": 480, "height_px": 854 } ] }
```
- 추천 키워드 없음(수동 `q`만). 414 방지 위해 `q` 길이 캡(gif 50 / image 100자).
- 결과 클릭 → `POST /assets/download`.

### 5.5 다운로드 / 업로드
```jsonc
// POST /assets/download  req { "url": "https://..." }   → resp = Asset
// POST /assets/upload    multipart file                 → resp = Asset
```
- 저장 후 이미지=치수 측정, 동영상=ffprobe(`width_px/height_px/duration_us/has_audio`).

### 5.6 동영상 분석 — 비동기 파이프라인
```jsonc
// 시작: POST /video-analysis (multipart)
//   fields: video(file), language, target_scene_number, apply_subtitle(bool)
{ "job_id": "vj_ab12" }                       // 즉시 반환

// 폴링: GET /jobs/:jobId
{ "job_id": "vj_ab12", "type": "video_analysis", "status": "running", "result": null }
// 완료
{ "job_id": "vj_ab12", "type": "video_analysis", "status": "done",
  "result": { "target_scene_number": 3, "scenes": [ /* 삽입할 Scene[] */ ] } }
```
**백엔드 job 처리 순서 (백그라운드)**
1. 업로드 영상 저장, job 생성(`pending`→`running`).
2. AI `POST /video-analysis/process` 호출 → 샷별 결과 수신.
3. **ffmpeg로 샷 경계 슬라이스** → 샷 클립 에셋(`source_type:"video_shot"`, ffprobe).
4. Scene[] 조립: 샷 1개 = 장면 1개. `media=클립`, `subtitle1Lines=[]`,
   `subtitle2Lines = apply_subtitle ? 분석줄 : []`, `voiceId=기본`, `fitToTts=false`.
5. `result={ target_scene_number, scenes }`, `status="done"`.
6. 프론트 폴링→완료→확인 모달(4.7)→`target_scene_number`에 삽입(§3 g).

**AI 내부 `POST /video-analysis/process`**
```jsonc
// req { "video_base64": "...", "language": "ko" }
// resp
{ "shots": [
  { "start_us": 0, "end_us": 1800000,
    "subtitle2_lines": [ { "text": "여러분 이거", "start_us": 0, "end_us": 900000 } ] }
] }
```
- **샷 경계**: Google VI `SHOT_CHANGE_DETECTION`만 사용(ko/ja 무관). 응답 `shot_annotations[].start_time_offset/end_time_offset`.
- **전사·번역**: 샷별 오디오를 ffmpeg로 추출 → **Gemini 오디오 입력**(전사+`language`로 번역을 한 번에, 소스 언어 무관) → **의미단위 분리** → `subtitle2_lines`.
- **자막2 타이밍**: 단어 타임스탬프 없음 → 샷 구간 `[start_us,end_us]` 내에서 문장(줄) **길이 비례 분배**(1.0 char-proportional fallback 방식 재사용).

**AI 내부 `POST /scenes/semantic-split`**
```jsonc
// req { "text": "...", "language": "ko" }   → resp { "lines": [ { "text": "..." } ] }
```

### 5.7 리소스 목록
```jsonc
// GET /templates → { "templates": [ { "name": "type1.png", "preview_path": "..." } ] }
// GET /samples   → { "samples":   [ { "name": "ref01.jpg", "path": "..." } ] }
// GET /styles    → { "styles":    [ { "name": "cinematic", "example_path": "..." } ] }
// GET /voices    → { "voices":    [ { "voice_id": "tc_voice_minsu", "name": "민수",
//                                     "preview_path": "voices/tc_voice_minsu.mp3" } ] }
```
- 백엔드가 `MY_*_DIR` 스캔. 스타일=`<name>.txt`+`<name>.jpg` 짝(텍스트는 비노출, `example_path`만).
- 보이스=`TYPECAST_VOICES` env 파싱, `preview_path`=`MY_VOICES_DIR/<voice_id>.mp3` 존재 시.

> 모든 키는 docker compose env로 주입.

### 6.1 Gemini (장면분할 · 의미분리 · 번역)
- 인증: `GEMINI_API_KEY`, 모델 `GEMINI_MODEL`(예 `gemini-2.5-flash`). **확정**
- SDK: `google-genai`, `genai.Client(api_key=...)`. JSON 강제: `response_mime_type="application/json"` + `response_schema`. **확정**
- 용도: `/scenes/split`(언어 분기), `/scenes/semantic-split`, 동영상 분석 내 번역.

### 6.2 Typecast (TTS)
- 인증: 헤더 `X-API-KEY: TYPECAST_API_KEY`. **확정**
- 엔드포인트: `POST https://api.typecast.ai/v1/text-to-speech/with-timestamps`. **확정**
- 파라미터: `voice_id`, `model=TYPECAST_MODEL`(예 `ssfm-v30`), `language`(`TYPECAST_LANG_MAP`: ko→kor, ja→jpn), `granularity="char"`, 속도 `TTS_SPEED`(audio_tempo), `output_format=TYPECAST_OUTPUT_FORMAT`(mp3). **확정**
- 2.0.0 변경: **다중 voice_id**(`TYPECAST_VOICES` 맵), 줄 단위 호출. 한 보이스가 ko/ja 모두 처리(언어 구분 안 함).

### 6.3 fal.ai (AI 미디어 — 4모델) — ✅ 재검증 완료 (2026-06)
- 인증: `FAL_KEY`. SDK: `fal-client`. **확정**
| 슬롯(env) | 확정 model id | 핵심 파라미터 | 비율 처리 |
|---|---|---|---|
| `FAL_MODEL_IMAGE_T2I` | `fal-ai/flux-pro/v1.1-ultra` | `prompt`, `aspect_ratio` | enum `…3:4,9:16,9:21` (9:16 ✓) |
| `FAL_MODEL_IMAGE_REF` | `fal-ai/nano-banana/edit` | `prompt`, `image_urls[]`, `aspect_ratio` | enum 포함 9:16 ✓ |
| `FAL_MODEL_VIDEO_T2V` | `fal-ai/wan-25-preview/text-to-video` | `prompt`, `aspect_ratio`, `resolution`, `duration` | `16:9\|9:16\|1:1` ✓ |
| `FAL_MODEL_VIDEO_I2V` | `fal-ai/wan-25-preview/image-to-video` | `prompt`, `image_url`, `resolution`, `duration` | ⚠️ **aspect_ratio 없음** |

- `resolution`(영상): `480p\|720p\|1080p`. `duration`: 문자열 `"5"\|"10"`.
- ⚠️ **I2V 비율 결정 변경**: i2v(wan·kling 동일)는 `aspect_ratio` 파라미터가 없고 **입력 이미지 비율을 그대로 따름**. 따라서 AI패널 비율 드롭다운은 **이미지·T2V에만** 적용, **동영상+참조 시엔 비활성/숨김**(출력=참조 비율, §A2(a) 중앙 구멍 전제와 일관).
- ⚠️ nano-banana 응답 `width/height` = **null 확정** → `value or 0` 가드 + 비율은 `aspect_ratio`로 강제(반환치 신뢰 금지).
- 참조 이미지 파라미터 차이: REF=`image_urls`(배열), I2V=`image_url`(단일).
- 대안(확정 live): T2I `fal-ai/flux-pro/v1.1`(단 `image_size` enum 사용), I2V `fal-ai/kling-video/v2.1/standard/image-to-video`(역시 비율 없음).

### 6.4 동영상 분석 — ⚠️ 재검증 결과: 설계 변경 필요
**핵심 발견 (확정)**: Google Video Intelligence의 `SPEECH_TRANSCRIPTION`은 **공식적으로 영어(en-US)만 지원**. 한국어/일본어 미지원이며, Google은 비영어 음성을 Cloud Speech-to-Text로 안내함. → **VI를 음성 전사에 쓸 수 없음.**
- VI `SHOT_CHANGE_DETECTION`은 ko/ja 무관하게 정상(샷 경계만). 응답 `shot_annotations[].start_time_offset/end_time_offset`(Duration). 과금 $0.05/분(라벨과 묶으면 무료), 무료 1,000분/월, inline base64(소형) 또는 GCS, async `operation.result()`. 패키지 `google-cloud-videointelligence`, 인증 `GOOGLE_APPLICATION_CREDENTIALS`.

**⚠️ 결정 5 (필요)** — 샷 감지 + ko/ja 전사·번역 조합:
- **(1) ffmpeg + Gemini (권장·최소)**: 샷 감지=ffmpeg scene detection, 전사·번역=Gemini 오디오 입력(소스 언어 무관, ko/ja 출력). GCP 불필요. 타이밍=샷/문장 비례 분배(단어 타임스탬프 없음).
- **(2) VI + Gemini (하이브리드)**: 샷 감지=Google VI(정확↑, GCP creds 필요), 전사·번역=Gemini.
- **(3) VI + Cloud Speech-to-Text + Gemini (정밀)**: STT가 ko/ja 단어 타임스탬프 제공하나 소스 언어 지정 필요·복잡·GCP 2API.

> ✅ **결정 5 (확정): (2) VI 샷 + Gemini 전사.** 샷 경계=Google VI `SHOT_CHANGE_DETECTION`(GCP creds 유지), 전사·번역=Gemini 오디오 입력(소스 언어 무관, ko/ja 출력). VI `SPEECH_TRANSCRIPTION` 미사용. 자막2 타이밍=샷 구간 내 문장 비례 분배(단어 타임스탬프 없음).

### 6.5 Giphy (gif 검색)
- 인증: `GIPHY_API_KEY`. 검색 `q`(50자 캡), `limit=GIPHY_LIMIT`. 응답 `data[].images.original/fixed_width`(width/height 문자열 → parseInt). **확정**

### 6.6 SerpAPI (이미지 검색)
- 인증: `SERPAPI_KEY`. Google Images 엔진, `q`(100자 캡). 응답 `images_results[].original/thumbnail/original_width/original_height`. **확정**

## §7 CapCut Export

> 골든 draft 스켈레톤을 클론해 가변부만 치환하는 **템플릿 기반 빌더**(1.0 계승). µs 타이밍, fps 30, 캔버스 1080×1920.

### 7.1 엔드포인트 / 산출물
```jsonc
// POST /export  req
{ "title": "강민경 횟집", "language": "ko", "template_id": "type1.png", "scenes": [ /* Scene[] */ ] }
// resp
{ "folder_name": "강민경 횟집", "draft_path": "<CAPCUT_DRAFT_ROOT>/강민경 횟집" }
```
- 산출 폴더(=`folder_name`) 아래: `draft_info.json` · `draft_meta_info.json` · `assets/` · `subtitles.srt`.
- **`CAPCUT_DRAFT_ROOT/<folder_name>` 에 직접 생성**(CapCut이 즉시 인식). 다운로드용 zip도 함께 제공.
- 에셋 경로 = `CAPCUT_DRAFT_ROOT` + `folder_name` + `assets/` (전부 env 기반, 호스트 경로 누출 0).

### 7.2 트랙 z-order (아래→위, 렌더 순서)
| 순서 | 트랙 | 내용 |
|---|---|---|
| 1 (최하) | **미디어** | 장면별 미디어 세그먼트(순차) |
| 2 | **프레임 오버레이** 🆕 | 선택 `template_id` 풀스크린, 전체 길이 (없으면 트랙 생략) |
| 3 | **자막1** | 내레이션 텍스트 |
| 4 (최상) | **자막2** | 원본음성 번역 텍스트 |
| (별도) | **오디오** | 자막1 줄별 TTS |

> 프레임이 미디어를 가려 "구멍 안에서만" 보이게 함(프레임 PNG 알파). 자막은 항상 프레임 위.

### 7.3 미디어 트랙
- 장면당 1 세그먼트. `target_timerange = { start: sceneStartUs, duration: durationUs }`.
- 풀스크린 cover(1.0 transform 동일). 정지(image/gif)·동영상 공통 구조.
- **동영상**: `material.type="video"`, `has_audio=true`, 세그먼트 `volume = muted ? 0 : 1`, `source_timerange`로 트림(`fitToTts`/샷 길이 기준).
- 각 세그먼트 **스캐폴드 번들**(speeds/canvases/placeholder_infos/material_animations/sound_channel_mappings/material_colors/vocal_separations) 생성 → `extra_material_refs`에 **fresh UUID**로 연결.

### 7.4 프레임 오버레이 트랙 🆕
- `template_id` 有: `MY_TEMPLATES_DIR/<template_id>`(PNG) → `assets/` 복사 → photo material 1개.
- 세그먼트: 풀스크린, `target_timerange = { start: 0, duration: totalDurationUs }`(전 장면 합).
- `template_id` null → 이 트랙 생략.

### 7.5 자막1 / 자막2 텍스트 트랙 (분리)
- **자막1**: `subtitle1Lines` 각 줄 = 텍스트 세그먼트, `target_timerange = [ lineStartUs, tts.durationUs ]`(§3 d). 위치=골든 스켈레톤의 **자막1 기본 위치**.
- **자막2**: `subtitle2Lines` 각 줄 = 세그먼트, `[ sceneStartUs+startUs, endUs-startUs ]`(§3 e). 위치=**자막2 기본 위치**.
- 텍스트 콘텐츠 빌드(폰트 `CAPCUT_FONT_PATH`). 해당 트랙에 줄 없으면 세그먼트 없음(빈 트랙 허용).
- 두 트랙의 화면 위치는 **골든 draft에 고정**(프레임은 데코일 뿐, 위치 메타데이터 아님 — §1.4 결정 1 취지).

### 7.6 오디오 트랙
- 자막1 줄별 TTS(`tts.localPath`) → 오디오 세그먼트, `target_timerange=[ lineStartUs, tts.durationUs ]`.
- 동영상 원본 오디오는 미디어 세그먼트 `volume`로 제어(7.3). 자막2엔 오디오 없음.

### 7.7 폴더명 규칙 (`title` → `folder_name`)
- **정제**: 금칙문자 `/ \ : * ? " < > |` 제거, 양끝 공백 trim. 빈 문자열 → `"clickclip"`.
- **충돌**: `CAPCUT_DRAFT_ROOT/<name>` 존재 시 `-2`, `-3` … 자동 접미사(비파괴).
- 최종 `folder_name`을 응답에 담아 화면 표시(§4.9).

### 7.8 SRT
- `subtitles.srt` = **자막1(내레이션) 기준** 누적 절대 타임코드.
- (자막2는 export 트랙에 포함되나 srt에는 미포함 — 단일 트랙 유지. 필요 시 추후 확장.)

### 7.9 무결성 / 메타
- 모든 material·segment·`extra_material_refs`는 **fresh UUID**(참조 무결성).
- `draft_meta_info.json`: 미디어 레지스트리(`metetype` photo/video, `draft_root_path=CAPCUT_DRAFT_ROOT`).
- `collectAssets`: workspace 에셋 + 프레임 PNG 복사 → `assets/` → 각 `capcutPath` 재작성.


## §8 UI/UX 원칙 (횡단)

> UI는 1.0 와이어프레임에 구속되지 않고 **사용성 우선**으로 CSS를 새로 짠다. 더 편한 구성이면 적극 채택.

1. **Compact 장면 카드**: 한 줄에 열을 늘리지 않는다. 자막1만 기본 노출, `ttsText`·자막2는 필요 시 펼침, 성우는 장면 기본값+줄별 예외.
2. **합성 미리보기**: 선택 장면을 미디어 + 프레임 + 자막 위치로 **합성**해 보여준다(결과 예측).
3. **로딩 = 애니메이션만**: 진행바·퍼센트 **금지**. **국소 표시**(전역 오버레이 금지), 작업 위치에만. 비동기 동영상 분석 중에도 다른 편집 가능.
4. **자막 색 매칭**: 자막1/자막2를 템플릿 프리뷰 영역 색과 매칭해 "어디로 나갈지" 직관화.
5. **컨텍스트 명확화**: 에셋 패널 상단 "장면 N 적용 중" + 선택 장면 하이라이트.
6. **파괴적 동작 가드**: 장면/자막 삭제·분리·병합은 즉시 반영하되 Undo/Redo로 복구(아래 범위).
7. **명시 버튼 + 단축키 병행**: 키보드 자막편집과 동일 동작의 버튼을 함께 제공(발견성).
8. **빈 상태·총 길이 상단 노출**: 장면 없음/에셋 없음 등 친절한 빈 상태, 총 길이 상시 표시.
